Generated by /office-hours on 2026-06-03

## Problem Statement

The micro:bit v2's built-in PDM microphone is hardwired into the CODAL audio
pipeline at construction time. Makers who want to use an external analog
microphone on an edge connector pin have no clean way to redirect the audio
pipeline to that source. Other MakeCode extensions that use audio (soundLevel(),
onSound(), audio-recording) continue to read from the built-in mic regardless.

The goal: a MakeCode extension that lets a maker swap the audio source to an
analog mic on a GPIO pin, with zero changes required to other extensions or
user code that uses the audio pipeline.

## What Makes This Cool

The CODAL audio pipeline's components (`levelSPL`, `processor`, `splitter`,
`rawSplitter`) are all public pointers on `MicroBitAudio`. That means an
extension can construct a parallel pipeline for any ADC-capable pin and
hot-swap those pointers at runtime. Code that calls `uBit.audio.levelSPL->getValue()`
— including pxt-microbit's own `soundLevel()` block — transparently reads from
the new mic without modification. Events fired via `DEVICE_ID_SYSTEM_LEVEL_DETECTOR`
come from the new mic automatically because a LevelDetectorSPL constructed with
that device ID fires on the same event bus channel.

One block. One pin picker. Everything downstream just works.

## Constraints

- micro:bit v2 only (nRF52833, CODAL runtime)
- MakeCode / pxt-microbit environment
- Must not require users of other audio extensions to modify their code
- Valid pins: ADC-capable edge connector pins (P0, P1, P2 confirmed; P3, P4,
  P10 also ADC-capable on nRF52833 but may conflict with other hardware)
- Swap must be called before any other audio blocks run (i.e., in "on start")

## Premises

1. C++ shims are required — TypeScript alone cannot access CODAL's NRF52ADC or
   uBit.audio internals.
2. The swap happens at user code runtime when the block is called, not at
   compile time or boot. User places it at the top of "on start".
3. Only ADC-capable edge connector pins are valid targets. The extension should
   document which pins are safe (P0, P1, P2 at minimum).
4. Transparency is achieved by replacing the public pointers on uBit.audio —
   not by rebuilding the CODAL runtime or forking pxt-microbit.
5. The built-in mic is disabled via `deactivateMic()` after the swap to prevent
   ADC contention.

## Approaches Considered

### Approach A: Pointer Swap (chosen)
Hot-swap the uBit.audio pipeline pointers at runtime. C++ shim creates a
parallel ADC→StreamSplitter→StreamNormalizer→LevelDetectorSPL chain for the
target pin, replaces the public pointers on uBit.audio, disables the built-in
mic. ~50 lines of C++. Single block in the MakeCode UI.

Effort: M (human: ~1-2 days / CC: ~30 min)
Risk: Medium — relies on uBit.audio fields staying public in future CODAL.

### Approach B: Level Injector
Poll the ADC pin from a fiber and push values into levelSPL. Rejected: cannot
reconstruct a proper audio stream, onSound() events won't work, audio-recording
extension won't redirect.

### Approach C: DataSource Wrapper
Custom AnalogMicSource class implementing CODAL's DataSource interface, inserted
as a switchable proxy at pipeline construction time. Cleanest architecture,
runtime switching between sources, but 3-4x more C++ and slower iteration.
Deferred to a future version if this extension gains real users.

## Recommended Approach: Pointer Swap (A)

**C++ shim (`analog-mic.cpp`)**:

```cpp
#include "pxt.h"

#if MICROBIT_CODAL
#include "MicroBitAudio.h"
#include "StreamSplitter.h"
#include "StreamNormalizer.h"
#include "LevelDetectorSPL.h"
#include "NRF52ADC.h"

namespace analogMic {

bool initialized = false;
volatile bool altMicActive = true;  // filterOn flag; set false to pause during deep sleep

//%
// gain: NRF52ADC hardware gain step, 0–7 (7 = built-in default; external modules
//       with onboard amp may clip — start at 0 and increase during bring-up)
// normFactor: StreamNormalizer normalisation factor, integer 0–1000 representing
//             0.000–1.000 (divided by 1000.0f before passing to CODAL).
//             Default 80 → 0.08f, matches built-in mic.
void useAnalogMicOnPin(int pinId, int gain, int normFactor) {
    if (initialized) return;  // Silent no-op if called twice — by design
    initialized = true;

    // Resolve AnalogPin enum to hardware pin.
    // pxt::getPin() is the correct pattern (confirmed from pxt-microbit/libs/core/pins.cpp:128).
    // On CODAL builds MicroBitPin IS NRF52Pin via typedef — cast is safe.
    NRF52Pin *targetPin = (NRF52Pin*) pxt::getPin((int)pinId);
    if (!targetPin) {
        target_panic(PANIC_HARDWARE_CONFIGURATION_ERROR);  // should never be null for valid AnalogPin
        return;
    }

    // Acquire an ADC channel for the analog pin.
    NRF52ADCChannel *altMic = uBit.adc.getChannel(*targetPin, false);
    if (!altMic) {
        target_panic(PANIC_HARDWARE_CONFIGURATION_ERROR);  // pin is not ADC-capable
        return;
    }

    altMic->setSampleRate(11000);  // CONFIG_AUDIO_DEFAULT_MICROPHONE_SAMPLERATE
    // gain 0–7; 0 = no additional gain (recommended for modules with onboard amp like MAX4466).
    // 7 matches built-in mic defaults — may clip with amplified modules.
    altMic->setGain(gain, 0);  // second arg = reference voltage selection, 0 = default

    // Build a parallel pipeline.
    // Order follows MicroBitAudio constructor: create splitter → attach stages → enable.
    // CODAL uses cooperative scheduling — no fiber yield here, so pointer swap is safe.
    StreamSplitter *altRawSplitter = new StreamSplitter(altMic->output);
    altRawSplitter->filterOn(&altMicActive);  // gates frames; set altMicActive=false to pause

    // StreamNormalizer(source, normFactor, activate, outputFormat, outputRate)
    float normFactorF = normFactor / 1000.0f;  // int promoted to float before division — no truncation
    StreamNormalizer *altProcessor = new StreamNormalizer(
        *altRawSplitter->createChannel(),  // channel 0 → processor
        normFactorF,
        true,
        DATASTREAM_FORMAT_8BIT_SIGNED,
        10);

    // Use generateDynamicID() to avoid CodalComponent registration collision with
    // the orphaned original splitter (which was also registered as DEVICE_ID_SPLITTER).
    // Callers use uBit.audio.splitter pointer directly — device ID is irrelevant to them.
    StreamSplitter *altSplitter = new StreamSplitter(
        altProcessor->output);  // default: generateDynamicID()

    // DEVICE_ID_SYSTEM_LEVEL_DETECTOR must match the original so that pxt-microbit's
    // onSound()/soundLevel() event registrations pick up events from this instance.
    LevelDetectorSPL *altLevelSPL = new LevelDetectorSPL(
        *altRawSplitter->createChannel(),  // channel 1 → levelSPL
        85.0, 65.0, 16.0, 35.0f,
        DEVICE_ID_SYSTEM_LEVEL_DETECTOR);

    // Disable original PDM mic: sets micEnabled=false, runmic=low, deactivates ADC channel.
    uBit.audio.deactivateMic();

    // Hot-swap all pipeline pointers, including mic so periodicCallback() tracks altMic
    // rather than the original PDM channel (prevents re-activation of built-in mic).
    uBit.audio.mic         = altMic;   // ← prevents periodicCallback() re-activation hazard
    uBit.audio.levelSPL    = altLevelSPL;
    uBit.audio.processor   = altProcessor;
    uBit.audio.splitter    = altSplitter;
    uBit.audio.rawSplitter = altRawSplitter;

    altMic->enable();
    // Note: micFilter (LowPassFilter*) declared in MicroBitAudio.h is NOT initialized
    // in MicroBitAudio's constructor (verified from CODAL source) — not in live path.
}

//%
void setAnalogMicEnabled(bool enabled) {
    altMicActive = enabled;
}

} // namespace analogMic
#endif
```

**TypeScript shim (`analog-mic.ts`)**:

```typescript
//% weight=100 color=#AA278F icon=""
//% block="Analog Mic"
namespace analogMic {
    /**
     * Use an analog microphone on an edge connector pin instead of the
     * built-in microphone. Call this at the start of your program before
     * any other sound blocks. Calling it more than once has no effect.
     *
     * Defaults (gain=7, normFactor=80) match the built-in mic behaviour exactly.
     * For modules with onboard amplification (e.g. MAX4466), start with gain=0.
     *
     * @param pin         analog-capable edge connector pin, e.g. AnalogPin.P1
     * @param gain        ADC hardware gain 0–7; default 7 matches built-in mic
     * @param normFactor  normalisation factor 0–1000 (÷1000 = 0.000–1.000); default 80 = 0.08
     */
    //% blockId=analog_mic_use_pin
    //% block="use analog microphone on pin %pin || gain %gain normFactor %normFactor"
    //% pin.fieldEditor="gridpicker"
    //% gain.min=0 gain.max=7 gain.defl=7
    //% normFactor.min=0 normFactor.max=1000 normFactor.defl=80
    //% expandableArgumentMode="toggle"
    //% group="Setup"
    export function useAnalogMicOnPin(
        pin: AnalogPin,
        gain: number = 7,
        normFactor: number = 80
    ): void {
        // shim implemented in analog-mic.cpp
    }

    /**
     * Pause or resume the analog microphone. Use before deep sleep to stop
     * the ADC sampling and reduce power consumption. Resume before reading
     * soundLevel() or using onSound() events.
     * @param enabled true to resume mic, false to pause
     */
    //% blockId=analog_mic_set_enabled
    //% block="set analog microphone enabled %enabled"
    //% group="Setup"
    export function setAnalogMicEnabled(enabled: boolean): void {
        // shim implemented in analog-mic.cpp
    }
}
```

The `||` in the `block` annotation is MakeCode's syntax for optional (expandable)
parameters. In the block editor, users see just the pin picker by default; clicking
`+` expands to reveal the gain (0–7) and normFactor (0–1000) inputs. Simple case
stays simple; advanced makers get the knobs.

**Parameter semantics**:

| Parameter | TypeScript default | C++ value passed | Meaning |
|---|---|---|---|
| `gain` | `7` | `7` | ADC hardware gain step; matches built-in mic |
| `normFactor` | `80` | `80` | Integer encoding of 0.08f (÷1000); matches built-in normaliser |

The C++ shim divides `normFactor` by `1000.0f` before passing to `StreamNormalizer`,
so the user-facing range of 0–1000 maps cleanly to CODAL's 0.000–1.000 float range
without requiring the block to expose floating-point inputs.

**Dependency note**: `"microphone": "*"` is intentionally excluded. The built-in
microphone extension initialises `uBit.audio.levelSPL` with the PDM mic during
startup; including it as a dependency risks the built-in pipeline running before
our swap. We access `uBit.audio` directly through CODAL — no pxt-microbit
microphone API is called.

**Hardware wiring**: A typical analog electret mic module (e.g., MAX4466-based)
connects VCC→3V, GND→GND, OUT→P1. The OUT pin is the analog audio signal.

## Open Questions

~~1. **LOOKUP_PIN macro**~~ — **RESOLVED**: use `pxt::getPin((int)pinId)` from `pxt-microbit/libs/core/pins.cpp:128`. Cast to `NRF52Pin*` is safe on CODAL builds (MicroBitPin IS NRF52Pin).

~~2. **micFilter**~~ — **RESOLVED**: `micFilter` declared in `MicroBitAudio.h` is NOT initialized in the constructor (verified from CODAL source). Not in the live pipeline path.

3. **Gain calibration** (first hardware session): Start with `gain=0` for modules
   with onboard amplification (e.g. MAX4466). Increase if `soundLevel()` stays
   near-zero. `gain=7` matches built-in mic defaults but will clip amplified modules.

~~4. **StreamSplitter constructor**~~ — **RESOLVED**: `StreamSplitter(DataSource&, uint16_t id = generateDynamicID())` confirmed from `codal-core/inc/streams/StreamSplitter.h`. `altSplitter` now uses `generateDynamicID()` (omit second argument) to avoid CodalComponent ID collision with orphaned original.

5. **audio-recording extension compatibility**: That extension calls
   `uBit.audio.splitter->createChannel()`. User must call our block before any
   audio-recording blocks run. Stale channel reference if called after is undefined.
   Future version could detect and warn.

## Success Criteria

- `soundLevel()` returns non-zero values from the analog pin mic when speaking
- `onSound(loud)` fires correctly when a loud sound is made near the analog mic
- The built-in mic no longer responds (deactivateMic() confirmed working)
- A second extension using `soundLevel()` works without modification when
  placed after the "use analog microphone on pin" block

## What I noticed about how you think

- You led with "the existing audio pipeline should work without modification" —
  that's an API-first constraint, not a feature request. Most people would have
  said "I want to use a different mic." You said "I want the system to be
  transparent." That's the harder, more interesting problem.
- You chose analog over I2S or PDM. That's the pragmatist's choice — simpler
  hardware, easier to breadboard, wider range of mic modules. It reveals you're
  building something to use, not something to show off.
- When asked about audience, you picked "makers wanting non-standard mics" over
  "educators" or "extension developers." That specificity matters — it means the
  API can be terse and technical rather than hand-held.
- You went straight to "extensible" as an option but didn't pick it. That's
  good instinct — extensible systems are great after you have a working concrete
  case, not before.

## NOT in Scope

- **I2S / PDM mic support**: Deferred. Analog via ADC is the v0 target. I2S requires a separate driver.
- **micFilter (LowPassFilter)**: Confirmed not in live pipeline — nothing to implement.
- **Automatic deep sleep integration**: Deferred. `setAnalogMicEnabled(false)` gives manual control; full CodalComponent lifecycle integration is a v1 concern.
- **Runtime mic switching (built-in ↔ analog)**: Deferred. The `initialized` guard is intentional — one-way swap for simplicity.
- **CI/CD pipeline**: Explicitly deferred for v0. Manual test + tag release.
- **Gallery submission process**: Deferred to after hardware validation.

## What Already Exists

- `uBit.audio` (MicroBitAudio): already has the full pipeline. We reuse every component class — no new driver code.
- `pxt::getPin()` in `pxt-microbit/libs/core/pins.cpp:128`: resolves pin IDs to hardware pins. Used directly.
- `StreamSplitter`, `StreamNormalizer`, `LevelDetectorSPL` from CODAL: reused as-is.
- `NRF52ADC::getChannel()`: already supports arbitrary ADC-capable pins.

## Failure Modes

| Codepath | Can fail? | Test covers it? | Error handling? | User sees? |
|---|---|---|---|---|
| `pxt::getPin()` returns null | Only for truly invalid pin values (not in enum) | No | target_panic() | Sad face — diagnosable |
| `getChannel()` returns null | Pin not ADC-capable | No | target_panic() | Sad face — diagnosable |
| Called after audio blocks | Stale channel reference | No | None | Silent audio breakage — documented limitation |
| `normFactor=0` | normalizer gain = 0.0f | No | None | soundLevel() always 0 — user error |
| Sound expression triggers activateMic() | periodicCallback re-activation | Partially (A1 fix) | uBit.audio.mic swap | Prevented by fix |
| Deep sleep without setAnalogMicEnabled(false) | Battery drain | No | filterOn flag | Increased power draw — documented |

**Critical gaps**: none after the A1 fix. The "called after audio blocks" case is a documented limitation, not a silent bug.
